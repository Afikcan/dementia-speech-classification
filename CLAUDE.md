## Overview

Dementia assessment from speech audio (.wav files) using fine-tuned transformer-based audio models. This is a course project (Instructor: Zongxing Xie) with three goals: (1) improve classification accuracy over a 37.5% baseline, (2) add explainability via attention maps, and (3) optimize for deployment via INT8 quantization.

The dataset is the **DementiaNet** corpus (455 .wav clips from 215 speakers: 84 dementia, 131 nodementia), organized by speaker into `dementia/` and `nodementia/` folders.

## Project History & Baseline

The original baseline (in `code_examples/`) used a **4-class** task (`no dementia`, `5 years to dementia`, `10 years to dementia`, `zero years to dementia`) with `wav2vec2-xls-r-300m` (300M params), 32-second random subsamples, and achieved **37.5% accuracy** on 32 test samples. The model mostly predicted "five years to dementia" and "no dementia," scoring 0% on "zero years to dementia." The original work ran on Google Colab with a V100 GPU.

The current pipeline has been **simplified to binary classification** (dementia vs nodementia), which is more tractable given the small dataset. The active scripts use smaller models (e.g., `wav2vec2-base`, 95M params) and 10-second clips. A new binary baseline has not yet been established.

## Data Pipeline

1. **Download**: `python download_audio.py` — fetches dementia/nodementia folders from Google Drive via gdown (>50 file limit per folder; may need manual zip download)
2. **Generate splits**: `python generate_csv.py` — scans `data/dementia/` and `data/nodementia/`, produces tab-separated `data/train_dm_new.csv` and `data/valid_dm_new.csv`
3. **Train**: `python train_model.py --model_type wav2vec2` — fine-tunes with HuggingFace Trainer

Data structure: `data/{dementia,nodementia}/PersonName/*.wav`. Splitting uses `GroupShuffleSplit` by person name to prevent data leakage (multiple clips per speaker).

## Training

```bash
python train_model.py --model_type wav2vec2 --epochs 5 --batch_size 4 --lr 3e-5
```

Supported `--model_type`: `wav2vec2` (default), `ast`, `hubert`, `whisper` (falls back to wav2vec2 — needs custom classification head). Audio is resampled to 16kHz and truncated/padded to 10s. Checkpoints saved to `./results/{model}-finetuned/`.

## Dataset Details

### File counts
- **On disk**: 131 dementia clips, 324 nodementia clips = **455 total**
- **New CSVs**: 359 train (110 dementia, 249 nodementia) + 96 valid (21 dementia, 75 nodementia) = 455 total. All paths verified to exist on disk.
- **No person leakage**: 0 speakers overlap between train and valid splits.
- **No held-out test set yet** — only train/valid. A 3-way split is needed for proper evaluation.

### Class imbalance (known issue)
The dataset is imbalanced: **28.8% dementia overall** (30.6% in train, 21.9% in valid). The legacy CSVs were more balanced (45.8% dementia) because they missed many nodementia files. This imbalance will bias models toward predicting "nodementia." Mitigation options:
- Class-weighted loss (`CrossEntropyLoss(weight=...)`)
- Oversampling the minority class
- Stratified splitting (current `GroupShuffleSplit` does not stratify by label)

### Legacy vs new CSVs
- `data/train_dm.csv` / `data/valid_dm.csv` = **legacy** (275 total, from original Colab notebooks, paths point to `/content/drive/My Drive/...` — won't work locally)
- `data/train_dm_new.csv` / `data/valid_dm_new.csv` = **active** (455 total, correct local paths, generated by `generate_csv.py`)
- The new CSVs picked up 180 more files that the legacy prep missed (mostly nodementia speakers)
- 2 legacy files (`ronano'rahilly_15_1`, `ronano'rahilly_15_2`) had apostrophes in filenames; on disk the folder is `Ronan O_Rahilly` (underscore), so the new CSVs correctly reference them as `ronano_rahilly_15_*`

### Other data files
- `data/DementiaNet - dementia.csv` — metadata for 84 dementia speakers (name, dementia type, birthdate, death, first symptoms year, YouTube URLs, gender, ethnicity, datasplit assignment). No equivalent metadata file exists for nodementia speakers.

## Architecture Notes

- `train_model.py` uses `AutoModelForAudioClassification` — clean but less control
- The original notebooks used a **custom `Wav2Vec2ForSpeechClassification`** class with mean pooling over hidden states and a custom classification head (dense → tanh → dropout → projection). This approach is needed for attention map extraction (`output_attentions=True`)
- The original also used a custom `DataCollatorCTCWithPadding` and `CTCTrainer` subclass

## Planned Improvements (Project Proposal)

1. **Multi-architecture comparison**: Wav2Vec2, AST, HuBERT, Whisper encoder
2. **Audio augmentation**: Critical given small dataset. Consider `audiomentations` library (time stretch, pitch shift, noise injection, SpecAugment). Multiple random crops per file also effectively multiplies data.
3. **Segment length experiments**: 10s vs 15s vs 20s vs 30s — longer clips capture more speech patterns (pauses, hesitations, word-finding difficulty)
4. **Explainability**: Attention map visualization over audio waveforms to show diagnostic "hotspots"
5. **Efficiency**: INT8 post-training quantization (`torch.quantization.quantize_dynamic`)
6. **Test set**: Current split is train/valid only — need a 3-way split (train/valid/test) for proper held-out evaluation
7. **Class imbalance**: Address the 71/29 nodementia/dementia ratio with weighted loss or oversampling
